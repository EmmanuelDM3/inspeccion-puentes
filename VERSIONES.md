# Registro de versiones del formulario

Este repositorio mantiene dos snapshots funcionales para presentación y trazabilidad:

- **Versión 1 (V1)**: estado base actual del sistema.
- **Versión 2 (V2)**: base de trabajo para implementar mejoras funcionales.

## Estructura

- `versiones/v1/`
  - Copia completa del formulario actual (`index.html`, `css/`, `js/`).
- `versiones/v2/`
  - Copia inicial del formulario sobre la que se aplicarán mejoras.

## Objetivo

Tener evidencia clara de evolución para mostrar al cliente final:

1. Qué se entregó inicialmente (V1).
2. Qué se está mejorando e implementando (V2).

## Cómo usar

Abrir directamente en navegador:

- `versiones/v1/index.html`
- `versiones/v2/index.html`

A partir de este punto, las mejoras se aplicarán en `versiones/v2/` para mantener intacta la referencia de `versiones/v1/`.
